# SOP — 会议记录整理

## 触发条件
老板开完会（通常来自 `Personal/Agenda/upcoming.md` 上排好的会），把原始纪要 / 录音转写交过来。
**直接进对应公司 `Meetings/`，不走 Inbox。**

## 前置
- 知道这场会属于哪家公司（不确定就问，绝不猜，避免串味 —— 铁律 #1）
- 若该会在 `agenda.md` 上，整理完把它从 agenda 移除（已发生）

## 步骤
1. 在对应公司 `Meetings/` 下，按 `YYYY-MM-DD-主题.md` 复制 `_template.md`。
2. 把原始内容粘到「原始记录」段，**不编辑**（铁律 #2）。
3. AI 从原始记录里拆出三股：
   - **决策** → 写进本会议的「决策」段，同时追加到 `Decisions/pending.md`，标来源 `[[本会议文件名]]`。
   - **任务** → 写进「任务」段，同时追加到 `Tasks/open.md`：标归属公司、负责人、截止；老板自己的进「待我处理」，分派别人的进「已分派」。
   - **待跟进/风险** → 留在本会议 + 必要时进当周 `Weekly/` 文件。
   - **下一场会 / 后续约定** → 进 `Personal/Agenda/upcoming.md`（未来的事）。
4. 写一句话总结。

## 产出
- 一份结构化会议记录（原始 + AI 三段产出）
- `Decisions/` 和 `Tasks/` 自动更新

## 检查清单
- [ ] 原始记录保留未改
- [ ] 每个决策、任务都标了来源会议链接
- [ ] 没有把别家公司的信息带进来
- [ ] 任务都有负责人和截止
